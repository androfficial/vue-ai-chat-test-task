# Internationalization (i18n) Guidelines

## Translation Files

Keep translations synchronized between languages:

**File Structure:**

```
locales/
├── index.ts    # i18n config and types
├── en.ts       # English translations
└── uk.ts       # Ukrainian translations
```

## Translation Keys

Use dot notation for nested keys:

```typescript
// locales/en.ts
export default {
  common: {
    save: 'Save',
    cancel: 'Cancel',
    delete: 'Delete',
  },
  chat: {
    new: 'New Chat',
    placeholder: 'Type a message...',
    send: 'Send',
  },
  settings: {
    title: 'Settings',
    api: {
      title: 'API Settings',
      apiKey: 'API Key',
    },
  },
}
```

## Usage in Templates

```vue
<template>
  <h1>{{ $t('settings.title') }}</h1>
  <v-btn>{{ $t('common.save') }}</v-btn>
</template>
```

## Usage in Script

```typescript
import { useI18n } from 'vue-i18n'

const { t, locale } = useI18n()

function getMessage() {
  return t('chat.placeholder')
}

function changeLanguage(lang: 'en' | 'uk') {
  locale.value = lang
}
```

## Pluralization

Use pipe syntax for plurals:

```typescript
messages: {
  en: {
    chat: {
      messageCount: 'no messages | 1 message | {count} messages'
    }
  }
}

// Usage
$t('chat.messageCount', { count: 5 }) // "5 messages"
```

## Adding New Translations

1. Add key to `en.ts`
2. Add same key to `uk.ts` with Ukrainian translation
3. Use in component with `$t('key')` or `t('key')`
4. Test both languages

## Type Safety

Define translation schema for TypeScript:

```typescript
// locales/index.ts
import type en from './en'

declare module 'vue-i18n' {
  export interface DefineLocaleMessage {
    typeof en
  }
}
```
